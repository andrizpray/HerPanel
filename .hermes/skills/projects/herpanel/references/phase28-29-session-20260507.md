## Critical Pitfalls Encountered

### 1. Patch Tool False "Found X Matches"
**Problem**: Patch tool reported "Found 4 matches" when editing `routes/web.php` and `Domains/Index.jsx` (files with repeated patterns).

**Fix**: Use Python I/O via `execute_code`:
```python
from hermes_tools import read_file, write_file
content = read_file('/path/to/file')['content']
# ... modify ...
write_file('/path/to/file', content)
```

### 2. web.php Corruption (Line Numbers)
**Problem**: File got corrupted with line numbers (`1|`, `2|`) appearing in actual file content. Caused `ParseError: syntax error, unexpected token "use"`.

**Fix**:
```bash
git checkout -- routes/web.php
# Then re-add changes carefully
sed -i '/use App\\Http\\Controllers\\SubdomainController;/a use App\\Http\\Controllers\\NewController;' routes/web.php
```

### 3. Duplicate Routes
**Problem**: Routes added twice — once with `/domains/` prefix and once without (from botched `execute_code` appends). `php artisan route:list` showed 12 routes instead of 6.

**Fix**: Check end of file for duplicate route groups:
```bash
tail -20 routes/web.php
# If duplicates found:
cat routes/web.php | head -n $(grep -n "require __DIR__.'/auth.php'" routes/web.php | cut -d: -f1) > routes/web_temp.php
mv routes/web_temp.php routes/web.php
```

### 4. Duplicate Imports
**Problem**: `use App\Http\Controllers\MimeTypeController;` appeared twice (line 8 and 9).

**Fix**: `sed -i 'Nd' routes/web.php` (where N = duplicate line number)

### 5. Terminal + PHP Inline Code Failure
**Problem**: Complex PHP with arrays/quotes failed in terminal `php artisan tinker --execute="..."`.

**Fix**: Use `execute_code` with Python to run PHP, or create temporary PHP script.
